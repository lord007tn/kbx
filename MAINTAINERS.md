# Maintainers

Current maintainer:

- `@lord007tn`

Maintainers are responsible for release approval, security advisory coordination, issue triage, and final decisions on project scope.

## Review Expectations

- Require passing CI before merging.
- Keep release-impacting changes tied to tests, docs, or a clear manual validation note.
- Treat changes under `.github/`, package publishing, dependency installation, MCP destructive tools, and session rewind behavior as security-sensitive.

## Security

Security reports should follow [SECURITY.md](./SECURITY.md). Do not request vulnerability details in public issues or pull requests.
