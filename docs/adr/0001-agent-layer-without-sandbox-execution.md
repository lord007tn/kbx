# Agent Layer Without Sandbox Execution

`kbx` will grow an optional agent helper layer on top of its local knowledge core, but it will not add a general-purpose sandboxed code execution tool in this roadmap. Sandboxed execution would change the product's security boundary and require command policy enforcement, runtime isolation, output caps, process cleanup, and hook interception; `kbx` will stay focused on knowledge operations such as search, refresh, source management, MCP tools, adapters, and agent guidance.
