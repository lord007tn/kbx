# Support

Use the channel that matches the type of help you need:

- Bugs: open a GitHub issue with reproduction steps, kbx version, Node.js version, OS, and relevant command output.
- Feature requests: open a GitHub issue and describe the workflow problem, expected behavior, and alternatives you considered.
- Usage questions: open a question issue with your environment and what you already tried.
- Security vulnerabilities: do not open a public issue. Follow [SECURITY.md](./SECURITY.md).

Clear reproduction details and small example workspaces make it much easier to respond.
